**Generating a PGP key at the Linux commandline** Created: 30 Dec 2020 [More articles like this](../index.html) Pretty Good Privacy (PGP) has been around forever. It has a reputation of being hard to use and even [experienced techies abandon it](https://arstechnica.com/information-technology/2016/12/op-ed-im-giving-up-on-pgp/) for it's original purpose of encrypting messages. As far as I can tell, though, it's the defacto standard for digitally signing code and binaries as authentic and is utilized by Github, despite some [embarrassing vulnerabilities](https://arstechnica.com/information-technology/2018/06/decades-old-pgp-bug-allowed-hackers-to-spoof-just-about-anyones-signature/?itm_source=parsely-api) exposed and fixed a couple of years ago. Requirements ============================================================== * email address * [GnuPG](https://gnupg.org/documentation/manuals/gnupg/) installed: For Debian/Ubuntu/PureOS/Kali and derivatives: ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~bash sudo apt-get install gpg ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ Generate a GPG Key ============================================================== Create a passphrase for your GPG Key beforehand and store in your password manager - unless you are creating a GPG Key for your password manager! In that case, make one up, and make it strong. ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ bash gpg --full-generate-key ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ If there is an error, try again, making sure you type your password correctly at the pop-up.
